# Flashcard App

A production-grade flashcard learning platform with spaced repetition, daily study sessions, quizzes, and progress tracking.

**Supported language pairs:** German ↔ Persian · German ↔ English · English ↔ Persian

---

## Tech Stack

| Layer     | Technology                                      |
|-----------|-------------------------------------------------|
| Frontend  | Vue 3, TypeScript, Pinia, Tailwind CSS          |
| Backend   | Ruby on Rails 8 (API-only), PostgreSQL, Redis   |
| Workers   | Sidekiq                                         |
| Auth      | JWT (HS256, 24h expiry) + bcrypt                |

## Backend Architecture

```
app/
├── controllers/api/v1/     # Thin controllers — orchestrate only
├── services/               # Business logic (dry-monads Result types)
│   ├── auth/
│   ├── decks/
│   ├── flashcards/
│   └── study/
├── queries/                # Read-only database queries
│   ├── decks/
│   ├── flashcards/
│   └── study/
├── policies/               # Authorization (manage? pattern)
├── serializers/api/v1/     # Response serializers
├── errors/                 # ApplicationError, AuthorizationError, NotFoundError
└── lib/
    └── json_web_token.rb
```

### Key Patterns

**Service objects** — all business logic, return `Success` / `Failure`:
```ruby
result = Decks::CreateDeckService.call(user: current_user, params: deck_params)
result.success? # => true / false
result.value!   # => the deck
result.failure  # => ["error message"]
```

**Query objects** — scoped reads, no mutation:
```ruby
decks = Decks::ListDecksQuery.call(user: current_user)
deck  = Decks::FindDeckQuery.call(user: current_user, id: params[:id])
```

**Serializers** — explicit response shaping:
```ruby
Api::V1::DeckSerializer.call(deck)
Api::V1::DeckSerializer.collection(decks)
```
