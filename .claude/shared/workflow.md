# workflow.md

Development Workflow:

Before starting:

1. Read CLAUDE.md
2. Read ARCHITECTURE.md
3. Read relevant skills
4. Review ROADMAP.md

Feature Development Flow:

1. Plan feature
2. Create backend structure
3. Create API endpoints
4. Create frontend integration
5. Write tests
6. Refactor
7. Update roadmap

Rules:

* Keep tasks small
* Avoid mixing unrelated changes
* Refactor after implementation
* Keep architecture consistent

Testing Workflow:

* Run backend tests
* Run frontend tests
* Run linting
* Fix warnings before finishing

Code Review Checklist:

* readable code
* scalable structure
* reusable logic
* no duplicated code
* proper naming
* proper testing

Important:
Production quality is required.
